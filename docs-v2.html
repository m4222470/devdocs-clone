Core Concepts

Understanding the fundamental concepts behind Devdocs X.

Architecture Overview

Devdocs X is built with a modular architecture that separates concerns and makes it easy to customize and extend.

Presentation Layer

HTML, CSS, JavaScript

Logic Layer

JavaScript Components

Data Layer

Content & Assets

Data Models

The system uses a hierarchical data model for organizing documentation content.

{
  "documentation": {
    "categories": ["getting-started", "api-reference", "guides"],
    "pages": {
      "getting-started": {
        "title": "Getting Started",
        "pages": ["introduction", "installation", "quick-start"]
      }
    }
  }
}

Component System

All UI components are built using reusable, composable patterns.

Buttons

Primary, Secondary, Ghost variants

Cards

Content containers with multiple styles

Navigation

Headers, sidebars, breadcrumbs

Forms

Inputs, selects, textareas

Theming System

Devdocs X supports theming through CSS custom properties.

:root {
  --primary-500: #6366F1;
  --primary-600: #4F46E5;
  --neutral-800: #1F2937;
  --font-sans: 'Inter', sans-serif;
}
